How the work was built

Method

The projects start with a practical judgment problem, then move through artifact building, boundary review, and private staging.

Working pattern

  • Frame the judgment problem in plain terms.
  • Build something concrete enough to test and explain.
  • Review claims, limits, source posture, and use conditions.
  • Stage privately before deciding what, if anything, should become public.

Reviewer note

Release gates and private/public status are tracked separately so the public-facing story can stay clear.
